Analysis, license & RNS.

1. Dataset analysis

Dataset: Accidentalidad 2025 (Traffic accidents, City of Madrid) — Madrid Open Data portal.

Schema: 

num_expediente, fecha, hora, localizacion, numero, cod_distrito, distrito, tipo_accidente, estado_meteorológico, tipo_vehiculo, tipo_persona, rango_edad, sexo, cod_lesividad, lesividad, coordenada_x_utm, coordenada_y_utm, positiva_alcohol, positiva_droga

Notes:

  • The dataset includes information on traffic accidents in Madrid, with fields such as accident case number, date and time, location (including UTM coordinates), district, accident type, vehicle and person type, demographics, and toxicology results. All values are stored as text.
  • After review, the following columns were determined to be irrelevant for our intended analysis and will be excluded: estado_meteorológico, cod_lesividad, and lesividad.

3. Resource Naming Strategy

Definition of the strategy for naming resources in the ontology and the data generated from the accident CSV.

  • URI Format

    • Use dereferenceable URIs that allow access to information about the resource:

    https://datos.madrid.es/ontology/Accident/{id}
  • URI Domain

    • Use institutional or trusted domains such as datos.madrid.es or gob.es to ensure reliability and persistence.

  • Path Structure

    • Organize resources hierarchically by type and year if applicable:

    https://datos.madrid.es/ontology/Accident/2025/ID12345
  • URI patterns
    • Resource Type URI Pattern
    Accident /Accident/{year}/{id}
    Vehicle /Vehicle/{id}
    Person /Person/{id}
    District /District/{id}
